---

If you've embedded the script but the widget is not visible, it's likely being blocked by security policies or configuration issues.

Pavy Widget outputs detailed error codes and troubleshooting links to your browser's Developer Tools (Console) when initialization fails.

## How to Diagnose

1. Open the webpage where the widget is installed.
2. Open **Developer Tools** in your browser (Windows: `F12` or `Ctrl+Shift+I`, Mac: `Cmd+Option+I`).
3. Select the **Console** tab.
4. Type `[Pavy]` in the filter box to see only Pavy-related logs.
5. Click the troubleshooting link provided next to the error message for a detailed guide.

## Troubleshooting Guides by Error Code

Refer to the guides below based on the error code displayed in the console.

| Error Code | Meaning | Quick Fix |
| :--- | :--- | :--- |
| **INVALID_API_KEY** | Invalid API Key | Verify your API key in the Admin panel. |
| **NOT_PUBLIC_KEY** | Secret Key Used | Use the Public Key starting with `pk_` instead of `sk_`. |
| **ORIGIN_NOT_ALLOWED** | Domain Not Whitelisted | Add your current domain to 'Allowed Domains' in the Admin panel. |
| **WORKSPACE_NOT_FOUND** | Workspace Issue | Ensure your workspace is active and not disabled. |
| **RATE_LIMITED** | Rate Limit Exceeded | Wait a moment. The SDK will retry automatically. |
| **NETWORK_ERROR** | Connection Failure | Check your internet or firewall settings. |
| **CORS_BLOCKED** | CORS Policy Blocked | Check browser security settings or proxy configurations. |

## Still Stuck?

If you've verified all settings and the widget still doesn't appear, check the following:

- **Z-Index Issues:** The widget might be hidden behind other UI elements.
- **Content Security Policy (CSP):** Ensure your website's CSP allows communication with Pavy servers (`*.pavy.ai`).
- **Browser Extensions:** Ad-blockers or script-blockers might be preventing the SDK from running.
