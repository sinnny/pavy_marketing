## 어떤 의미인가요?
CORS(Cross-Origin Resource Sharing)는 브라우저가 보안을 위해 다른 도메인으로의 요청을 제한하는 정책입니다. Pavy 서버가 현재 귀하의 도메인을 신뢰할 수 있는 도메인으로 인식하지 못할 때 발생합니다.

## 확인 방법
개발자 도구의 **Console** 또는 **Network** 탭에서 `Access-Control-Allow-Origin` 관련 에러 메시지가 있는지 확인하세요.

## 해결 방법
1. **도메인 등록:** 대부분의 경우 [어드민 > 도메인 관리]에 도메인을 등록하지 않아 발생합니다. [ORIGIN_NOT_ALLOWED 가이드](./origin-not-allowed)를 참조하여 도메인을 등록하세요.
2. **프로토콜 확인:** `http`와 `https`는 다른 도메인으로 간주됩니다. 운영 환경이라면 반드시 `https`를 사용하고 도메인도 `https://`를 포함하여 등록해야 합니다.

![DevTools Console의 CORS 에러 메시지 예시 캡처](/images/troubleshoot/cors-blocked-devtools.svg)

## 여전히 해결되지 않나요?
로컬 개발 환경(`localhost`)에서 테스트 중이라면, 어드민에 `http://localhost:3000` 등을 명시적으로 추가했는지 확인해 보세요.
