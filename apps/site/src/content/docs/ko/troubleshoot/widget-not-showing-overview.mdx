웹사이트에 스크립트를 삽입했음에도 위젯이 보이지 않는다면, 보안 정책이나 설정 문제로 인해 초기화가 차단되었을 가능성이 높습니다. 

Pavy 위젯은 문제 발생 시 브라우저 개발자 도구(Console)에 상세한 에러 코드와 해결 방법을 출력합니다.

## 진단 방법

1. 위젯이 설치된 웹페이지를 엽니다.
2. 브라우저에서 **개발자 도구**를 엽니다 (Windows: `F12` 또는 `Ctrl+Shift+I`, Mac: `Cmd+Option+I`).
3. **Console(콘솔)** 탭을 선택합니다.
4. 필터 입력창에 `[Pavy]`를 입력하여 Pavy 관련 로그만 확인합니다.
5. 출력된 에러 메시지와 함께 표시된 링크를 클릭하여 상세 가이드를 확인하세요.

## 에러 코드별 해결 가이드

표시된 에러 코드에 따라 아래 가이드를 참조하세요.

| 에러 코드 | 의미 | 해결 방법 요약 |
| :--- | :--- | :--- |
| **INVALID_API_KEY** | 잘못된 API 키 | 어드민에서 API 키를 다시 확인하세요. |
| **NOT_PUBLIC_KEY** | 시크릿 키 사용 | `sk_` 대신 `pk_`로 시작하는 퍼블릭 키를 사용하세요. |
| **ORIGIN_NOT_ALLOWED** | 도메인 미등록 | 어드민의 '도메인 관리'에 현재 도메인을 추가하세요. |
| **WORKSPACE_NOT_FOUND** | 워크스페이스 문제 | 워크스페이스가 활성화 상태인지 확인하세요. |
| **RATE_LIMITED** | 요청 제한 초과 | 잠시 후 다시 시도하세요. (자동 재시도됨) |
| **NETWORK_ERROR** | 통신 장애 | 네트워크 연결 상태를 확인하세요. |
| **CORS_BLOCKED** | 보안 정책 차단 | 브라우저 보안 설정 또는 프록시 설정을 확인하세요. |

## 여전히 해결되지 않나요?

모든 설정을 확인했음에도 위젯이 보이지 않는다면 다음 사항을 추가로 점검해 보세요.

- **Z-Index 문제:** 위젯이 다른 UI 요소 뒤에 숨겨져 있을 수 있습니다.
- **Content Security Policy (CSP):** 웹사이트의 CSP 설정이 Pavy 서버(`*.pavy.ai`)와의 통신을 차단하고 있는지 확인하세요.
- **브라우저 확장 프로그램:** 광고 차단 프로그램이 스크립트 실행을 막고 있을 수 있습니다.

도움이 필요하시면 [고객 지원팀](mailto:support@pavy.ai)으로 문의해 주세요.
